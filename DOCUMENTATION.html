Présentation du projet

Vue d'ensemble & objectifs

Contexte

APISAHI Group est un groupe lifestyle africain réunissant 4 marques complémentaires dédiées à l'art de vivre, au bien-être, à la beauté naturelle et aux expériences cadeaux personnalisées. Ce site web constitue la vitrine numérique officielle du groupe.

Le projet a été développé from scratch (à partir de zéro) en React avec Vite, en suivant les contenus fournis par le client dans un document PDF de référence.

Objectifs du site

🎯
Présenter le groupe
Exposer la vision, la mission et les valeurs d'APISAHI Group ainsi que ses 4 filiales.
💼
Vitrine commerciale
Détailler les offres et packs de services de chaque filiale avec tarification.
📸
Galerie visuelle
Mettre en valeur les créations et réalisations via une galerie filtrée et immersive.
📩
Génération de leads
Capturer des demandes de contact et devis via un formulaire optimisé.

Architecture technique

Routes & pages

Le site est une SPA (Single Page Application). La navigation entre les pages se fait sans rechargement du navigateur grâce à React Router DOM v6. Chaque changement de route déclenche une animation de transition (fade + slide via Framer Motion).

/

Page d'accueil — HomePage

Slider hero plein écran (4 slides avec autoplay), section d'introduction du groupe avec 4 valeurs, grille des 4 filiales sur fond noir, bannière CTA dorée.

/groupe

APISAHI Group — GroupPage

PageHero 40vh, présentation textuelle illustrée, section Mission/Vision/Valeurs sur fond noir (3 colonnes), timeline des 4 filiales avec icônes et liens vers leurs offres.

/univers

Nos Univers — UniversPage

PageHero + grille 2×2 de cartes flip 3D (effet CSS perspective). Face avant : icône + nom + tagline. Face arrière (au survol) : description + bouton CTA coloré selon la filiale.

/offres

Nos Offres & Services — OffersPage

Navigation sticky avec onglets (défile avec la page), 4 sections filiales alternant fond beige/blanc, chaque section affiche les packs en grille avec la carte centrale mise en avant (badge "Recommandé"). Scroll vers ancre au chargement si URL contient #zeya, #lalou, etc.

/galerie

Galerie Photos/Vidéos — GalleryPage

Boutons de filtre par catégorie (Tout voir / ZEYA / LALOU / Senteurs / Rituel / Événements), grille masonry animée avec AnimatePresence (les items se réorganisent visuellement lors du filtrage), lightbox plein écran au clic avec navigation clavier (← → Escape).

/contact

Contactez-nous — ContactPage

Formulaire de contact complet (validation côté client, toast de confirmation), informations de contact (téléphone, WhatsApp, e-mail), liens réseaux sociaux (Instagram, Facebook, LinkedIn, TikTok).

/senteurs

Senteurs d'Afrique — SenteursPage AUTONOME

Page one-page dédiée avec sa propre navbar (liens ancres), footer, écran de chargement et 6 sections : Hero carousel (3 slides), Introduction, Valeurs, Produits e-commerce (9 produits), Galerie, Contact. Fond noir/doré — charte Senteurs d'Afrique.

/rituel

Rituel de Beauté — RituelPage AUTONOME

Page one-page dédiée avec sa propre navbar (boutons arrondis), footer, écran de chargement et 6 sections : Hero carousel (3 slides), Introduction, Valeurs, Soins & Produits (9 produits), Galerie, Contact. Fond plum/rose — charte Rituel de Beauté.

⚙️

Système de routes autonomes : Dans App.jsx, le tableau STANDALONE_ROUTES = ['/senteurs', '/rituel'] masque la Navbar et le Footer globaux APISAHI Group sur ces routes, et sélectionne l'écran de chargement adapté à chaque marque.

💡

Le composant ScrollToTop réinitialise automatiquement la position de scroll à 0 à chaque changement de route ET affiche un bouton flottant doré pour remonter en haut de page après 400px de défilement.

Architecture des composants

Hiérarchie globale

# Arbre de rendu — système routes autonomes (v2)
<App>
  <BrowserRouter>
    <Layout>                 # Lit useLocation() pour détecter la route
      <LoadingComponent />   # APISAHI | SenteursLoading | RituelLoading
      <ScrollToTop />
      {!isStandalone && <Navbar />}    # Masqué sur /senteurs et /rituel
      <AnimatePresence mode="wait">
        <Routes>
          # Routes APISAHI Group (avec Navbar + Footer globaux)
          <Route path="/"        element={<HomePage />} />
          <Route path="/groupe"  element={<GroupPage />} />
          <Route path="/univers" element={<UniversPage />} />
          <Route path="/offres"  element={<OffersPage />} />
          <Route path="/galerie" element={<GalleryPage />} />
          <Route path="/contact" element={<ContactPage />} />
          # Routes AUTONOMES (Navbar + Footer propres à la marque)
          <Route path="/senteurs" element={<SenteursPage />} />
          <Route path="/rituel"   element={<RituelPage />} />
        </Routes>
      </AnimatePresence>
      {!isStandalone && <Footer />}    # Masqué sur /senteurs et /rituel
      <Toaster />
    </Layout>
  </BrowserRouter>
</App>

Données & fonctionnalités

Hooks personnalisés

useScrollAnimation

Encapsule react-intersection-observer pour déclencher des animations Framer Motion dès qu'un élément entre dans le viewport.

// Utilisation dans n'importe quel composant
const { ref, inView, variants, staggerVariants } = useScrollAnimation(0.15)

return (
  <motion.div
    ref={ref}
    variants={staggerVariants}
    initial="hidden"
    animate={inView ? "visible" : "hidden"}
  >
    <motion.p variants={variants}>Je s'anime au scroll</motion.p>
  </motion.div>
)

Retourne : ref (à attacher à l'élément), inView (booléen), variants (fade+translateY 40px), staggerVariants (délai 150ms entre enfants). L'animation est déclenchée une seule fois (triggerOnce: true).

useGalleryFilter

Gère l'état du filtre actif et retourne les items filtrés.

const { active, setActive, filtered } = useGalleryFilter(GALLERY_ITEMS)
// active: string (id catégorie), filtered: tableau d'items
// Si active === 'all', retourne tous les items

Données & fonctionnalités

Fonctionnalités clés

Formulaire de contact

Champs : Nom complet, E-mail, Téléphone, Service souhaité (select), Message, Case de confidentialité.

Validation côté client : nom requis, regex e-mail, message requis, case cochée. Les erreurs s'affichent sous chaque champ.

Soumission : Sans backend (site vitrine). La soumission simule un délai de 1,2s (spinner), puis affiche un toast doré : "Merci ! Votre message a bien été envoyé. Nous vous répondrons sous 48h."

⚠️

Pour connecter un vrai backend, il suffit de remplacer le setTimeout dans ContactForm.jsx par un appel fetch() ou axios.post() vers votre endpoint API.

Lightbox galerie

Rendue via ReactDOM.createPortal() directement dans document.body pour éviter les conflits de z-index. Navigation :

  • Clic sur une image → ouvre la lightbox
  • Touches ← → → navigation entre images
  • Touche Escape → fermeture
  • Clic sur l'overlay sombre → fermeture

Navigation sticky avec ancres (page Offres)

Un IntersectionObserver surveille chaque section filiale. Lorsqu'une section dépasse 30% de visibilité dans le viewport, l'onglet correspondant dans la barre sticky devient actif (souligné doré). Le lien /offres#zeya depuis d'autres pages déclenche automatiquement le scroll vers la section correspondante.

Déploiement & maintenance

Intégration des images

Le site est actuellement fonctionnel avec des placeholders visuels (emojis + dégradés). Pour intégrer les vraies photos :

1

Préparer les images

Optimiser toutes les photos à moins de 200 Ko chacune (format WebP recommandé). Outils : Squoosh, TinyPNG, ImageOptim.

2

Déposer dans public/images/

Respecter l'arborescence : public/images/hero/hero-1.jpg, public/images/gallery/gallery-01.jpg, etc.

3

Mettre à jour HeroSlider.jsx

Remplacer les dégradés CSS par de vraies images : ajouter backgroundImage: 'url(/images/hero/hero-1.jpg)' dans le style de chaque slide.

4

Mettre à jour gallery.js

Vérifier que chaque src dans GALLERY_ITEMS correspond au bon nom de fichier déposé.

5

Remplacer les placeholders dans GalleryGrid.jsx

Remplacer les div à fond dégradé par des balises <img src={item.src} alt={item.alt} />.

Déploiement en production

Option recommandée — Vercel (gratuit)

1

Pousser le code sur GitHub

Créer un dépôt GitHub, ajouter le remote, commiter et pousser tout le projet (sauf node_modules/ et dist/).

2

Créer un compte Vercel

Se connecter sur vercel.com avec le compte GitHub. Cliquer "New Project" et importer le dépôt APISAHI Group.

3

Configuration Vercel

Framework preset : Vite. Build command : npm run build. Output directory : dist. Laisser le reste par défaut.

4

Déployer

Cliquer "Deploy". Vercel génère une URL en .vercel.app. Chaque push sur la branche main redéploie automatiquement.

5

Connecter un nom de domaine personnalisé

Dans Settings → Domains, ajouter apisahigroup.com (ou votre domaine). Vercel configure automatiquement HTTPS.

Autres options d'hébergement compatibles

Netlify

GitHub Pages
Nécessite vite.config.js avec base: '/repo-name/'. Via GitHub Actions.
Hébergement classique
FTP du contenu de dist/. Configurer le serveur pour rediriger toutes les routes vers index.html.
⚠️

Important pour les hébergeurs classiques (Apache/Nginx) : étant une SPA, toutes les URLs (ex. /galerie) doivent pointer vers index.html. Pour Apache, créer un fichier .htaccess dans dist/. Pour Nginx, utiliser try_files $uri /index.html.

Pages autonomes — Senteurs d'Afrique & Rituel de Beauté
